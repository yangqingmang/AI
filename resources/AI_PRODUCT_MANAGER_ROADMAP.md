# BAT P7+ 级 AI 产品经理 (AI PM) 专家路线图

**目标**: 成为能主导 AI 业务落地、具备深厚技术同理心（Technical Empathy）的产品专家。
**核心能力**: 技术边界认知 + 数据策略 + 商业 ROI + 评估体系构建。
**工具箱**: Python (用于数据分析/原型验证), Excel/SQL, Prompt Engineering, 飞书/钉钉文档。

---

## 阶段一：技术祛魅 —— 理解大模型的“脾气” (1 个月)
**目标**: 能够自信地与算法 P8 对话，不提“伪需求”。

### 1. 核心概念深度理解 (Concept Deep Dive)
*   **Transformer 原理 (PM 版)**:
    *   不需要懂矩阵乘法，但要懂 **Attention 机制**（模型在关注什么？）。
    *   要懂 **Context Window (上下文窗口)**：为什么长文本会贵？为什么会遗忘？
    *   要懂 **Temperature / Top-P**: 如何通过参数控制模型的“创造力” vs “稳定性”。
*   **Token Economics (代币经济学)**:
    *   **实战任务**: 建立一个 Excel 模型，计算“假设日活 100万，每次对话消耗 500 Token，使用 GPT-4 vs Qwen-72B vs Llama-3-8B 的成本差异”。这是 PM 必须算得清的账。

### 2. 提示词工程 (Prompt Engineering) 进阶
*   **超越“你可以帮我吗”**:
    *   掌握 **CoT (Chain of Thought)**: 让模型通过“也就是”来提升逻辑能力。
    *   掌握 **Few-Shot Prompting**: 给示例比给指令更有效。
    *   掌握 **Structured Output**: 强制模型输出 JSON，以便工程对接。
*   **实战**: 在你的 `enterprise-brain` 项目中，优化 `ingest.py` 或 `query.py` 里的 Prompt，目标是让“幻觉率”降低 20%。

## 阶段二：架构决策 —— RAG vs Fine-tuning (1.5 个月)
**痛点**: 老板问“我们要不要训练自己的大模型？”，你要能给出专业判断。
**目标**: 掌握 AI 落地的两种核心范式。

### 1. RAG (检索增强生成) —— 产品的短期外挂
*   **理解**: 向量数据库 (Vector DB) 是什么？它为什么有时候搜不到（语义匹配 vs 关键词匹配）？
*   **PM 视角**:
    *   如何设计“引用来源”功能来增加用户信任？
    *   如何处理“切片 (Chunking)”策略？（切太细丢失上下文，切太粗噪音太大）。

### 2. Fine-tuning (微调) —— 产品的长期内功
*   **理解**: SFT (监督微调) 能改变说话风格，但很难补充新知识。
*   **决策模型**:
    *   什么时候用 RAG？(需要实时性、知识库巨大)。
    *   什么时候微调？(需要特定格式输出、需要降低 Prompt 长度以省钱、需要私有化部署)。

## 阶段三：数据飞轮与评估体系 (Data Strategy & Eval) (2 个月)
**这是 P7 PM 与 P6 PM 的分水岭。** P6 关注功能上线，P7 关注效果迭代。

### 1. 构建评估集 (Golden Set / Benchmark)
*   **任务**: 不要只看代码跑通。你需要建立一套**“黄金测试集”**。
    *   收集 50 个真实用户的刁钻问题。
    *   定义好坏标准：准确性、相关性、安全性。
*   **自动化评估**: 了解 **LLM-as-a-Judge** (让 GPT-4 当裁判给小模型打分) 的方法论。

### 2. 数据飞轮设计 (Data Flywheel)
*   **核心逻辑**: 用户的每一次交互（点赞、点踩、修改答案），如何变成下一次训练的数据？
*   **实战**: 设计一个 UI 交互方案，诱导用户对 AI 的回答进行反馈（Feedback），并设计后台逻辑如何存储这些反馈。

---

## 阶段四：Agent 与未来交互 (1.5 个月)
**目标**: 探索 Chat 之外的交互形态。

### 1. Agent (智能体) 认知
*   **理解**: 什么是 **Function Calling (工具调用)**？模型怎么知道该去查天气还是查库存？
*   **场景**: 设计一个“企业助理 Agent”。它不仅能回答问题，还能**代表用户执行动作**（比如：帮我请个假 -> 调用 HR 系统 API）。

### 2. 交互设计 (UX for AI)
*   **Latency Masking**: AI 回答慢怎么办？(流式输出、骨架屏、有趣的 Loading 文案)。
*   **Ambiguity Handling**: 当用户说不清需求时，AI 应该怎么反问？(设计“澄清式对话”流程)。

## 阶段五：实战作品集 (PM Portfolio)

**你的简历不应该堆砌技术名词，而应该展示“产品决策文档”。**

1.  **Project A: 企业知识库 PRD (产品需求文档)**
    *   基于 `enterprise-brain`，写一份包含**架构图、用户旅程图 (User Journey Map)、数据流转图**的 PRD。
    *   重点章节：成本预估、幻觉风险控制方案。

2.  **Project B: 垂直领域模型 ROI 分析报告**
    *   假设你要做“法律 AI 助手”。写一份报告，分析是租用 OpenAI API 划算，还是自己租卡部署开源模型划算？临界点在哪里？

3.  **Project C: 评测报告 (Evaluation Report)**
    *   选取 3 个模型（如 Qwen, GPT-4, Claude-3），在特定任务上进行横向评测，输出可视化对比图表。
